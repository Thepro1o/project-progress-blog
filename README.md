# 📊 Project Progress Blog

A beautiful, simple blog website to track your year-long project progress. Share updates, milestones, challenges, and learnings all in one place!

## ✨ Features

- 📝 **Add Progress Updates** - Create posts with title, date, category, and content
- 🏷️ **Categorize Posts** - Organize posts into categories (Update, Milestone, Challenge, Learning, Other)
- 🔍 **Filter by Category** - View posts by specific categories
- 💾 **Auto-Save** - Posts are automatically saved to your browser's local storage
- 📱 **Responsive Design** - Works perfectly on desktop, tablet, and mobile devices
- 🎨 **Beautiful UI** - Modern design with gradient colors and smooth animations
- 🗑️ **Easy Deletion** - Delete individual posts or clear all posts at once

## 🚀 Getting Started

### Option 1: Use GitHub Pages (Recommended)
1. Go to your repository settings
2. Scroll to "GitHub Pages" section
3. Select "main" branch as the source
4. Your site will be live at: `https://Thepro1o.github.io/project-progress-blog`

### Option 2: Run Locally
1. Clone the repository
2. Open `index.html` in your web browser
3. Start adding your progress updates!

## 📖 How to Use

1. **Fill in the form:**
   - Enter a title for your update (e.g., "Week 1 - Setup Complete")
   - Select the date
   - Choose a category
   - Write your progress update

2. **Click "Add Post"** - Your post will appear at the top of the blog

3. **Filter posts** - Use the dropdown to view posts by category

4. **Delete posts** - Click the delete button on any post to remove it

## 📚 Data Storage

Your posts are saved in your browser's **Local Storage**, which means:
- ✅ Data persists between sessions
- ✅ No server or database needed
- ⚠️ Data is stored per browser/device (use GitHub/cloud sync for cross-device access)

### Optional: Sync Across Devices
To share data between computers:
1. Export your posts from one computer
2. Import them to another
3. Or use GitHub's sync features with a backend (future enhancement)
